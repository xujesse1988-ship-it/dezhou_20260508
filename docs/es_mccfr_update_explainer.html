一次 ES-MCCFR update 是怎么走完的

把一次训练更新想成一条带探照灯的路线：本次被训练的玩家会在自己的每个决策点把所有选择都试一遍；对手节点只按当前策略抽一条路。走到终局后，收益一路回传，更新 regret 和平均策略。

谁被更新 traverser = update_count % 2，两名玩家轮流成为本次 update 的主角。
哪里全展开 只有 traverser 自己的节点会枚举所有动作，计算每个动作的 counterfactual value。
哪里采样 对手节点按当前策略抽一个动作；NLHE 的发牌随机性已经在 root 一次性完成。
输入 当前 regret 表、strategy_sum 表、随机数源、bucket table。
路径 从 root 递归到 terminal，traverser 节点全展开，对手节点采样。
输出 一批 regret delta 和 strategy_sum 增量，最后 update_count += 1。
安全网 动作来自 legal_actions；真正 apply 时规则层还会拒绝非法金额。

步骤导航

1. 选 traverser

一次 update 只更新一个玩家。heads-up 下，update 0 更新玩家 0，update 1 更新玩家 1，之后继续轮流。这样每个玩家都会从自己的视角学习：哪些动作比当前混合策略更好，哪些更差。

traverser = update_count % n_players update_count: 0 -> 1 -> 2

traverser 节点：像做选择题复盘

轮到本次要训练的玩家时，算法不会只抽一个动作，而是把 Fold、Call、Raise 等所有合法抽象动作都试一遍。每条分支递归到底，得到每个动作的价值，再和当前混合策略的平均价值比较。

对手节点：像按概率抽一张路线卡

轮到对手时，算法按对手当前策略分布采样一个动作，只沿这一条路继续。这样一次 update 的成本会低很多，但长期大量采样后，仍然能估计整棵博弈树的方向。

当前策略 sigma(a) = max(R(a), 0) / sum(max(R, 0))；如果全非正，就均匀随机。
节点价值 sigma_value = sum_a sigma(a) * cfv(a)，表示当前混合策略的平均价值。
regret 增量 delta(a) = cfv(a) - sigma_value；更好的动作涨 regret，更差的动作降 regret。
和 NLHE 适配层的关系： 本页讲的是训练器的一次 update。落到 `SimplifiedNlheGame` 时，`info_set` 会把手牌/公共牌 bucket、位置、stack、下注状态、街道和动作金额语义签名打包；`legal_actions` 从规则层合法动作集合生成默认 action 抽象；`next` 再把抽象动作转成规则层动作并 apply。
