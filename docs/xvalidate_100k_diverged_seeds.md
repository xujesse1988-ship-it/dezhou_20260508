## 关键观察

1. **桶 B 形状高度同质**：95 条全部落在 `{−1,+1}` 或 `{−1,−1,+2}` 两个 multiset 上，无离群 — bug 局限于「pot 余 chip 累计 / 偏置方向」局部逻辑，而非更广义的分配错误。
2. **B-3way 的 +2/−1/−1 模式**：单一赢家拿到 +2，自然解释是**多个 side pot 的余 chip 全堆同一 button-左邻**。与 D-039-rev1「整笔给 button 左侧最近的获胜者」一致，但 PokerKit 显然在多 pot 间采取不同累积策略。
3. **桶 A 全是 2-人 swap**：所有 10 条均为 `(a, b) → (b, a)` 形式的两人摊牌顺序倒置。无 3 人或更长序列错乱 — 起点选择 bug 概率高于整体方向 bug。
4. **chip-conservation 全部满足**：B 的 95 条 deltas sum=0，没有引入 / 销毁 chip — 漏的是分配，不是核算。
